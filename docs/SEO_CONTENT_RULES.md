# SEO Blog Content Development Rules & Requirements

## Master SEO Checklist for Blog Articles

### 1. RESEARCH & ANALYSIS PHASE
Must be completed before writing.

#### 1.1 Topic Research
- [ ] Read 5+ competitor articles on the topic
- [ ] Review Wikipedia page (if exists) for comprehensive overview
- [ ] Check Google snippets and featured snippets for the topic
- [ ] Identify content gaps (what's missing from competitors)
- [ ] Note unique angles/perspectives
- [ ] Map out search intent (informational/commercial/transactional)

#### 1.2 Keyword Analysis
- [ ] Primary keyword identified and intent verified
- [ ] LSI keywords extracted (related/long-tail variations)
- [ ] Semantic keywords identified
- [ ] Commercial intent check if applicable
- [ ] Target audience identified
- [ ] Keyword difficulty assessed

#### 1.3 Competitor Analysis
- [ ] Top 3 competitor articles analyzed
- [ ] Their structure mapped
- [ ] Their word count noted (aim to match or exceed)
- [ ] Unique content angles identified
- [ ] Link sources identified

---

### 2. STRUCTURE REQUIREMENTS

#### 2.1 Heading Hierarchy (MANDATORY)
- **H1**: Article title (keyword included, <60 chars)
- **H2**: Main sections (5-8 sections)
  - **H3**: Subsections under each H2 (2-3 per H2)
- **H4**: Sub-subsections only if necessary
- ❌ Never skip heading levels (no H1→H3 jumps)

#### 2.2 Essential Sections
Every article MUST include:
1. **Introduction** (150-200 words)
   - Hook/opening that captures attention
   - Clearly state what reader will learn
   - Include primary keyword naturally
2. **Main Content** (4-6 H2 sections covering topic thoroughly)
3. **FAQ Section** (5-8 questions with concise answers)
4. **Conclusion** (100-150 words)
   - Summarize key points
   - Call-to-action (CTA)
   - Related resources link

#### 2.3 Content Types to Include
At least 3 of these:
- Lists (unordered or numbered)
- Tables (comparison or data)
- Images/diagrams (with alt text)
- Example scenarios
- Case studies
- Quotes/expert insights
- Code blocks (if technical)
- Blockquotes from sources

---

### 3. KEYWORD & SEO REQUIREMENTS

#### 3.1 Keyword Density
- Primary keyword: **1-2% density** of total words
- Natural placement (NOT keyword stuffed)
- Variation check required
  - Use plural forms
  - Use long-tail variations
  - Use LSI keywords
  - Use semantic synonyms

#### 3.2 Keyword Placement (PRIORITY LOCATIONS)
✓ Must be in:
- H1 title
- First paragraph (within 100 words)
- At least 1 H2 heading
- Meta title
- Meta description

✓ Should be in:
- Alt text (at least 1 image)
- Internal link anchor text

#### 3.3 LSI & Semantic Keywords
- Include 5+ LSI keyword variations
- Use natural language variations
- Include related terms
- Check for semantic relevance

#### 3.4 Keyword Stuffing Indicators
❌ RED FLAGS (Avoid):
- Keyword appears >3% of word count
- Unnatural keyword placement
- Keyword in every sentence
- Keyword in headings without context
- Keyword in anchor text unnatural

---

### 4. CONTENT QUALITY REQUIREMENTS

#### 4.1 Word Count
- **Minimum**: 1,800 words (preferably 2,200+)
- **Target**: 2,500-3,500 words for depth
- **Exception**: Some quick-answer articles can be 1,200+
- Distribute evenly across sections

#### 4.2 Readability Standards
- **Flesch Reading Ease**: 50-70 (conversational level)
- **Grade Level**: 8-10 (accessible to most readers)
- **Sentence length**: Avg 15-20 words
- **Paragraph length**: Max 150 words per paragraph

#### 4.3 Language & Tone
Requirements:
- [ ] Professional but friendly tone
- [ ] Active voice (>85% of sentences)
- [ ] Vary sentence structure
- [ ] Use contractions (it's, we're) for conversational feel
- [ ] Define technical terms on first mention
- [ ] No jargon without explanation
- [ ] Show expertise (E-E-A-T factor)

#### 4.4 Passive Voice Check
- Maximum 10-15% passive voice
- Convert constructions like:
  - "The spice was used..." → "Chefs use the spice..."
  - "It is believed..." → "Experts believe..."

---

### 5. LINK REQUIREMENTS

#### 5.1 Internal Links (ON-PAGE)
- **Minimum**: 3 links
- **Optimal**: 5-8 links
- Rules:
  - Point to relevant articles on site
  - Use descriptive anchor text (not "click here")
  - Link early (within first 1000 words if possible)
  - Link to product pages naturally (when contextual)
  - Maximum 1-2 links per 100 words

#### 5.2 External Links (REFERENCES)
- **Minimum**: 3-5 external references
- Requirements:
  - Link to authoritative sources only
  - Use "nofollow" or normal depending on trust
  - Cite fact-based sources
  - Use properly formatted citations
  - Include Wikipedia links if relevant

#### 5.3 Link Anchor Text
✓ GOOD:
- "Read our guide to [topic]"
- "Learn more about [specific term]"
- Keyword-rich but natural

❌ AVOID:
- "Click here"
- Over-optimization ("best [keyword] [keyword]")
- Same anchor text repeated >3 times

---

### 6. E-E-A-T (Expertise, Experience, Authority, Trustworthiness)

#### 6.1 Expertise Signals
- Demonstrate deep knowledge
- Include specific data/statistics
- Reference authoritative sources
- Show methodology/reasoning
- Use expert language (not dumbed down)

#### 6.2 Experience
- Include personal examples where relevant
- Share case studies
- Include "I've learned" or "in my experience"
- Show practical application

#### 6.3 Authority
- Cite credible sources
- Include expert opinions
- Reference industry standards
- Show the article is updated/current

#### 6.4 Trustworthiness
- [ ] Fact-check all claims
- [ ] Include source citations
- [ ] Be transparent about limitations
- [ ] Disclose affiliate links if present
- [ ] Include author bio (if known)
- [ ] Show last updated date

---

### 7. ON-PAGE SEO TECHNICAL

#### 7.1 Meta Tags
- **Meta Title**: 50-60 characters, keyword included
- **Meta Description**: 150-160 characters, compelling, includes CTA

#### 7.2 URL Slug (Pre-planning)
- Lowercase, hyphenated, 3-5 words
- Include primary keyword
- Example: `/blog/best-kerala-spices-for-cooking`

#### 7.3 Image Optimization
- At least 1 image per 500 words
- Alt text: descriptive, keyword-relevant
- Image format: WebP or JPEG (optimized)
- File size: <100KB (compressed)
- Dimensions: 600x400px minimum

#### 7.4 Schema Markup
Include structured data:
- `Article` schema (required)
- `Author` schema
- `FAQPage` schema (for FAQ section)
- `BreadcrumbList` schema

---

### 8. CONTENT VERIFICATION CHECKLIST

After writing, verify:
- [ ] Primary keyword appears 15-30 times (for 2500-wd article)
- [ ] Keyword in H1, first paragraph, at least 1 H2
- [ ] Keyword density 1-2%
- [ ] No keyword stuffing
- [ ] 5+ LSI keywords used
- [ ] 5-8 H2 sections
- [ ] At least 2-3 H3 per H2
- [ ] FAQ section present (5+ Q&A)
- [ ] 3-8 internal links
- [ ] 3+ external references
- [ ] Flesch score 50-70
- [ ] Active voice >85%
- [ ] No passive voice >15%
- [ ] Images with alt text
- [ ] Word count 1800+
- [ ] Readability verified
- [ ] Links working
- [ ] CTA present
- [ ] Mobile-friendly formatting
- [ ] No plagiarism (0% copied)

---

### 9. COMMON MISTAKES TO AVOID

❌ **Keyword Issues**
- Keyword not in title or first paragraph
- Too low density (<1%)
- Too high density (>3%)
- Keyword stuffing
- Ignoring search intent

❌ **Structure Issues**
- No clear H2/H3 hierarchy
- Inconsistent heading formatting
- No table of contents
- No FAQ section
- No internal links
- No CTA

❌ **Content Issues**
- Too short (<1800 words)
- Copied from competitors
- No original research/insights
- Poor readability
- Lots of jargon
- No source citations
- Outdated information

❌ **Technical Issues**
- Bad meta tags
- Broken links
- No alt text on images
- Too many images (slows load)
- No schema markup
- Not mobile-optimized

---

### 10. QUALITY SCORING SYSTEM

Each article is scored 0-100:

| Component | Points | How Evaluated |
|-----------|--------|---------------|
| Keyword Optimization | 15 | Density, placement, variations |
| Content Quality | 20 | Originality, depth, accuracy |
| Readability | 15 | Flesch score, sentence variety |
| Structure | 20 | Headings, sections, hierarchy |
| Links | 15 | Internal/external, quality |
| E-E-A-T | 15 | Expertise signals, citations |

**Minimum Score**: 70/100 to publish
**Target Score**: 85/100+

---

### 11. MULTI-PASS REVIEW PROCESS

#### Pass 1: Research & Planning (Step 1-2)
- Gather intelligence
- Plan structure
- Identify angles

#### Pass 2: Draft Writing (Step 6)
- One continuous pass, don't edit yet
- Focus on capturing ideas
- Aim for word count

#### Pass 3: AI Review (Step 7)
- Fact accuracy
- Tone consistency
- Optimization check

#### Pass 4: Issue Identification (Step 8)
- Run 50+ quality checks
- List all issues
- Prioritize by severity

#### Pass 5: Redevelopment (Step 9)
- Fix critical issues
- Improve density/readability
- Enhance structure
- Add missing content types

#### Pass 6: Final Polish (Step 10)
- Manual review
- Final fact check
- Brand voice alignment
- Ready to publish

---

### 12. FAQ SECTION REQUIREMENTS

Minimum 5 questions, organized by:
1. **Definitional** (What is...?)
2. **How-to** (How to...?)
3. **Why** (Why is...?)
4. **Comparison** (Which is better...?)
5. **Objection Handling** (Common concerns)

Each answer should be:
- 2-3 sentences
- Include relevant keyword naturally
- Link-friendly (internal links when applicable)

---

### 13. CONTENT TIMELINE & UPDATES

- **New articles**: Published as-is from pipeline
- **First review**: 2 weeks after publish (check rankings, engagement)
- **Update triggers**:
  - Ranking drop >5 positions
  - CTR <2%
  - Keyword trend changes
  - Competing content improves
  - New research/data available

---

## IMPLEMENTATION IN CONTENT GENERATION ENGINE

The 10-step pipeline enforces these rules by:
1. **Step 1**: Researching to meet 1.1-1.3
2. **Step 2**: Creating structure to meet 2.1-2.3
3. **Step 3**: Verifying compliance with 2.1-2.3
4. **Step 4**: Planning internal links per 5.1
5. **Step 5**: Gathering references per 5.2
6. **Step 6**: Writing content following 3.1-3.4, 4.1-4.4
7. **Step 7**: AI review for 4.2-4.4, 6.0
8. **Step 8**: Runs automated checks against sections 3, 4, 5
9. **Step 9**: Fixes identified issues iteratively
10. **Step 10**: Final manual review

Each article must pass ALL requirements in Section 8 (Content Verification Checklist) before publication.
