# AnnaSEO — Content Freeze & Blog System

Lifecycle-managed blog production: generate → approve → freeze → schedule → publish → monitor.

---

## Blog Lifecycle

```
generate  →  draft (body generated by ContentGenerationEngine)
approve   →  approved (human review complete)
freeze    →  frozen (scheduled_date assigned, no more edits without unfreeze)
publish   →  published (pushed to WordPress via ruflo_publisher, published_url set)
```

---

## Tables

### content_blogs (freeze system)

```sql
blog_id, project_id, pillar, cluster, keyword, title, body, meta_desc,
schema_type, language, status, frozen_at, scheduled_date, published_url,
internal_links, word_count, seo_score, version, created_at, updated_at
```

### blog_versions

Snapshot on every body edit. `version` int in content_blogs increments each time.

```sql
id, blog_id, version, title, body, meta_desc, saved_at
```

### content_freeze_queue

Lookup table for upcoming scheduled publications.

```sql
id, blog_id, project_id, pillar, scheduled_date, published (0|1), published_at
```

## Date Distribution Algorithm

When `POST /api/blogs/{project_id}/{blog_id}/freeze` is called without a `scheduled_date`:

1. Walk forward from today, day by day (max 400 days)
2. Count existing frozen blogs for this `(project_id, pillar)` on each candidate date
3. Assign the first date with fewer than **20 blogs** for this pillar
4. If `scheduled_date` is provided explicitly, use it directly

This ensures even distribution: **10–20 blogs per pillar per day** across the 1-year calendar.

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/blogs/{project_id}/generate` | Generate blog from brief (keyword + pillar) |
| POST | `/api/blogs/{project_id}/generate-batch` | Background: generate N blogs for a pillar |
| GET  | `/api/blogs/{project_id}` | List blogs (filter: pillar, status, date range) |
| GET  | `/api/blogs/{project_id}/stats` | Counts by status and pillar |
| GET  | `/api/blogs/{project_id}/calendar` | Blogs grouped by scheduled_date |

| GET  | `/api/blogs/{project_id}/freeze-queue` | Upcoming scheduled publications |
| GET  | `/api/blogs/{project_id}/due-today` | Blogs scheduled for today, status=frozen |
| GET  | `/api/blogs/{project_id}/{blog_id}` | Single blog |
| PUT  | `/api/blogs/{project_id}/{blog_id}` | Update title/body/meta (saves version snapshot) |
| DELETE | `/api/blogs/{project_id}/{blog_id}` | Delete blog |
| POST | `/api/blogs/{project_id}/{blog_id}/approve` | Set status=approved |
| POST | `/api/blogs/{project_id}/{blog_id}/freeze` | Set status=frozen, assign scheduled_date |

| POST | `/api/blogs/{project_id}/{blog_id}/publish` | Publish via ruflo_publisher |
| POST | `/api/blogs/{project_id}/{blog_id}/regenerate` | Re-generate body (keeps keyword/title) |
| GET  | `/api/blogs/{project_id}/version-history/{blog_id}` | All version snapshots |
| PUT  | `/api/blogs/{project_id}/{blog_id}/schedule` | Change scheduled_date |
| POST | `/api/blogs/{project_id}/populate-calendar` | Create blog stubs from strategy priority_queue |

## Internal Links

Before publishing, `internal_links` (JSON array) is injected into the body. These links come from the strategy `internal_link_map` (pillar → supporting article URLs). Injection happens in the `publish` route before calling `ruflo_publisher`.

## Frontend: BlogCalendarPage

Located at `/blogs` in the nav (App.jsx).

### Tabs

| Tab | Content |
|-----|---------|
| Calendar | 12-month grid: rows=pillars, cells=blog counts by month; click to drill |
| Upcoming Queue | Next 7 days of frozen blogs |
| Generate Batch | Form: select pillar + count → background generation |
| Due Today | Blogs scheduled today; "Publish now" button per item |

### Blog editor modal

Opens when clicking any blog title. Fields: title, body (textarea), meta_desc.
- **Save** — `PUT /api/blogs/{id}/{blog_id}` + creates version snapshot
- **Regenerate** — `POST /api/blogs/{id}/{blog_id}/regenerate`
- **Version history** — lists all previous versions with restore option

### Populate from Strategy

"Populate from Strategy" button calls `POST /api/blogs/{id}/populate-calendar` which reads the latest strategy session's `priority_queue` and creates draft blog stubs with pre-assigned scheduled dates spread across 12 weeks.

## Daily Publish Job

`GET /api/blogs/{project_id}/due-today` returns blogs where:
- `status = 'frozen'`
- `scheduled_date = today`

A cron job (or manual trigger) calls `POST /api/blogs/{id}/{blog_id}/publish` for each. The publish route:
1. Injects internal links into body
2. Calls `ruflo_publisher.publish(blog)` → WordPress REST API
3. Sets `status='published'`, `published_url`, submits URL to IndexNow
